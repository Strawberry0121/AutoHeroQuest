# AutoHeroQuest

## ゲーム概要
「AutoHeroQuest」は、勇者が自動で挑む冒険ゲームです。
Enterキーを押すだけで50回の分岐イベントが進行し、戦闘や仲間との協力、回復、宝探し、罠など多彩な出来事が登場。勇者や仲間のライフポイントを管理しつつ進み、
主人公のHPが0になると冒険は即終了。最終スコアで冒険の成否が決まり、800点以下はゲームオーバー。毎回異なる展開で、新鮮な自動冒険体験を楽しめます。

## 操作方法
- ゲーム開始時に **Enterキー** を押すだけで勇者が自動で行動します。
- リザルト画面で **リトライ** か **ゲーム終了** を選択可能です。

## プロジェクト構成（Eclipse用）

## 仕様
- 分岐イベント：50回
- イベント種類：戦闘、仲間、回復、宝、罠
- ライフポイント管理（主人公・仲間）
- 前回と同じイベントは連続せず新鮮な展開
- スコア3000点以下はゲームオーバー
- Enterキー押下で自動進行

## 開発環境
- Java 21
- Spring Boot 4.0.2
- Eclipse IDE
- Gradle

## Q&A（ChatGPTに聞いた内容まとめ）

**Q1. プロジェクト名は何にする？**  
**A1.** 「AutoHeroQuest」に決定。自動進行（Auto）、勇者（Hero）、冒険（Quest）を表現。

**Q2. メインクラス `Main.java` はどこに書く？**  
**A2.** `src/main/java/com/example/AutoHeroQuestApplication.java` に書く。Spring Boot メインクラスとして起動。

**Q3. 1回の操作でゲームを進めたい場合は？**  
**A3.** Enterキー押下で勇者が自動的にイベントを50回分進行する仕様に変更。

**Q4. 分岐やイベントの作り方は？**  
**A4.** Eventクラスを作り、戦闘・仲間・回復・宝・罠のイベントリストからランダム選択。前回と同じイベントは除外。

**Q5. 戦闘時の演出はどうする？**  
**A5.** Eventクラスに `List<String> actions` を持たせ、ランダムに技や逃走演出を表示。

**Q6. ライフポイントやスコア管理は？**  
**A6.** GameModelで主人公LP、仲間LP、スコアを管理。主人公LPが0でゲーム終了。スコアで冒険の成否判定。

**Q7. Spring Boot プロジェクトでテストエラーが出る**  
**A7.** 原因はテスト依存関係が不正。`spring-boot-starter-test` のみで解決。テストクラスはメインと同じパッケージに置く。

**Q8. Eclipseでの設定は？**  
**A8.** Gradle → Refresh、JDK21指定、テストクラスのパッケージ確認。必要に応じてプロジェクト削除→再インポート。

**Q9. 分岐は何回？**  
**A9.** 50回。毎回イベントをランダム選択、前回と同じイベントは連続しない。
